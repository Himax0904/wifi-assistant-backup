**版本：** v3.0  
**更新日期：** 2026-03-03  
**核心变化：** Knowledge + Tech 合并为 Knowledge Tech（知识进化官）

---

## 快速开始

### 默认模式（推荐）
直接描述任务，系统自动匹配角色：

```
"写一份项目周报"              → Writer
"调研一下竞品情况"            → Researcher
"帮我安排这周的工作"          → OPS
"写个脚本处理这个数据"        → Developer
"生成一张产品宣传图"          → Designer
"整理这个项目的知识库"        → Knowledge Tech
"把这次经验沉淀下来"          → Knowledge Tech
```

### 显式调用模式
```
/writer 写一份...
/research 调研一下...
/ops 安排一下...
/dev 写个代码...
/design 生成一张...
/kt 整理一下...
```

## 角色能力详解

### 🖋️ 笔杆子（Writer）

**核心能力：** 文档撰写、内容编辑、格式输出

**典型任务：**
- 工作周报/月报
- 会议纪要整理
- 方案文档撰写
- 内容润色优化
- PPT 大纲编写

**输出特点：** 结论先行，结构化，简洁直接

---

### 🧠 参谋（Researcher）

**核心能力：** 信息搜集、调研分析、决策支持、**结构化输出**

**典型任务：**
- 市场调研
- 技术方案对比
- 竞品分析
- 政策法规查询
- 风险评估
- **Data Table 数据对比表** ✨
- **思维导图梳理** ✨

**输出特点：** 信息来源标注，事实与观点分离，结构化格式优先

**NotebookLM 增强：** 支持 Data Table、引用溯源、多源整合

---

### 📊 运营官（OPS）

**核心能力：** 任务管理、流程设计、协调监控

**典型任务：**
- 工作优先级安排
- 项目进度跟踪
- 流程 SOP 设计
- 待办事项管理
- 风险预警

**输出特点：** 清晰时间线和责任人，可执行的行动项，状态可视化

---

### 🔧 研发官（Developer）

**核心能力：** 代码开发、工具构建、系统集成

**典型任务：**
- 脚本开发
- API 集成
- Bug 修复
- 性能优化
- 技术方案设计

**输出特点：** 代码注释清晰，使用说明完整，风险点标注，可回滚方案

---

### 🎨 设计师（Designer）

**核心能力：** 图片、PPT、视频视觉创作

**典型任务：**
- AI 绘图（文生图）
- PPT 设计与制作
- 短视频创作
- 海报、宣传图设计
- 图片编辑优化
- 数据可视化图表
- **Slide Deck 自动生成** ✨
- **Infographic 信息图** ✨

**输出特点：** 文件格式明确，分辨率/尺寸标注，多版本选择，视觉美感优先

**NotebookLM 增强：** 可调用 NotebookLM Slide Deck/Infographic 能力

---

### 🧬 知识进化官（Knowledge Tech）✨ 合并角色

**核心能力：** 知识管理 + 技能进化 + NotebookLM 集成

**典型任务：**

**📚 知识管理：**
- 多源文档整理与归档
- 知识结构化、标签化
- 知识库建设与检索优化
- **NotebookLM 文档处理** ✨

**📊 结构化输出：**
- **Data Table 数据表** ✨
- **Mind Map 思维导图** ✨
- **Briefing Doc 简报** ✨
- **Study Guide 学习指南** ✨
- **Flashcards 抽认卡** ✨
- **Quiz 测验题** ✨

**🧬 技能进化：**
- 项目复盘与经验沉淀
- 技能文档编写与更新
- 工作流优化
- 新工具评估引入

**输出特点：** 结构化格式优先，引用溯源标注，可检索可复用，版本记录完整

**使用场景：**
```
"整理这个项目的知识库"
"把这些文档做成结构化报告"
"生成竞品对比数据表"
"创建学习抽认卡"
"用 NotebookLM 处理这份文档"
"把这次经验沉淀成 Skill"
"优化我们的工作流"
```

## 角色协作关系

### Knowledge Tech 与其他角色

| 协作方 | 协作内容 |
|--------|----------|
| Researcher | 调研结果 → 知识库沉淀 |
| Writer | 文档内容 → 结构化归档 |
| Designer | 视觉素材 → 知识库存储 |
| Developer | 工具开发 → 知识文档化 |
| OPS | 流程优化 → SOP 固化 |

### NotebookLM 协作流程

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   MAX 输入   │ →   │  无牙分析   │ →   │Knowledge Tech│
│  任务/文档   │     │  任务拆解    │     │  NotebookLM │
└─────────────┘     └──────────────┘     └─────────────┘
                                                │
                                                ↓
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  最终输出   │ ←   │  无牙整合   │ ←   │  结构化    │
│  给 MAX     │     │  多源信息    │     │  输出结果   │
└─────────────┘     └──────────────┘     └─────────────┘
```

## 多角色协作示例

### 示例 1：竞品分析报告

```
任务："做个竞品分析报告，要数据对比表"

流程：
1. Researcher → 搜集竞品信息
2. Knowledge Tech → 生成 Data Table 对比表
3. Writer → 撰写分析报告
4. Designer → 制作可视化图表
5. Knowledge Tech → 沉淀到知识库
```

### 示例 2：项目文档整理

```
任务："把这 10 份项目文档整理成知识库"

流程：
1. Knowledge Tech → 上传 NotebookLM 处理
2. Knowledge Tech → 生成 Briefing Doc + Mind Map
3. Writer → 补充说明文字
4. Knowledge Tech → 归档到飞书知识库
```

### 示例 3：学习材料制作

```
任务："帮我理解这份技术文档"

流程：
1. Knowledge Tech → 上传 NotebookLM
2. Knowledge Tech → 生成 Learning Guide + Flashcards
3. Researcher → 补充技术背景
4. 交付学习材料包
```

### 示例 4：技能沉淀

```
任务："把这次项目经验沉淀下来"

流程：
1. Knowledge Tech → 项目复盘
2. Knowledge Tech → 经验提炼
3. Knowledge Tech → 编写 Skill 文档
4. Knowledge Tech → 版本管理与归档
```

## 最佳实践

1. **任务描述要清晰** — 越具体，匹配越准确
2. **复杂任务拆分成步骤** — 便于多角色协作
3. **重要结果要验收** — 总指挥把关，你最终决策
4. **定期复盘沉淀** — Knowledge Tech 持续优化体系
5. **敏感文档不上传** — NotebookLM 仅限非敏感内容
