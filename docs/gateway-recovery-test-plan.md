# Gateway 自动恢复测试方案

**测试目标**: 验证看门狗能否在 Gateway 故障时自动检测并恢复

**测试时间**: 2026-03-04 21:36

---

## 测试步骤

### 1. 记录初始状态
- [ ] 记录当前 Gateway PID
- [ ] 确认看门狗正常运行
- [ ] 确认消息通道正常

### 2. 模拟故障
- [ ] 手动停止 Gateway 进程
- [ ] 记录停止时间

### 3. 观察恢复
- [ ] 等待看门狗检测（最多 60 秒）
- [ ] 记录检测时间
- [ ] 记录重启时间
- [ ] 记录新 PID

### 4. 验证结果
- [ ] Gateway 进程恢复
- [ ] 端口监听恢复
- [ ] 消息发送正常
- [ ] 告警消息已发送

---

## 成功标准

| 指标 | 目标 | 实际 |
|------|------|------|
| 检测时间 | < 60 秒 | - |
| 恢复时间 | < 120 秒 | - |
| 告警发送 | 成功 | - |
| 服务恢复 | 正常 | - |

---

## 回滚方案

如果自动恢复失败：
```bash
# 手动重启 Gateway
systemctl --user start openclaw-gateway

# 验证状态
systemctl --user status openclaw-gateway
```
