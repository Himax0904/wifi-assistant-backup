## Token 使用管理

### 预算分配
- **总预算：** 根据当前余量合理分配
- **日均：** 总预算 ÷ 3
- **优先级：** 核心任务优先，探索性任务次之

### 使用原则
- 节约优先，避免浪费
- 复杂任务可调用子代理
- 简单任务本地处理
- 定期监控余量

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Token 不足 | 任务中断 | 优先核心任务，暂停探索 |
| API 注册失败 | 集成延迟 | 备用方案，本地处理 |
| 角色冲突 | 效率下降 | OPS 协调，总指挥决策 |
| NotebookLM 限制 | 功能受限 | 能力内化，降低依赖 |
