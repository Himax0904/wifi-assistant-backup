## 🎯 核心原则

**短任务一次性汇报，长任务分阶段汇报**

---

## 📏 任务分类与汇报方式

### 🟢 短任务（< 15 秒）→ 一次性汇报

**判断标准**：
- 单次工具调用
- 简单查询操作
- 文件读写（小文件）
- 状态检查

**汇报流程**：
```
1. 收到任务 → 立即确认 "收到，正在处理..."
2. 执行任务
3. 完成后 → 一次性输出完整结果
```

**示例**：
```
用户：检查 Gateway 状态
无牙：收到，正在检查 Gateway 状态...

（5 秒后）

无牙：✅ Gateway 运行正常
   - 状态：active (running)
   - PID: 123456
   - 运行时长：2 小时
```

---

### 🟡 中长任务（15 秒 - 2 分钟）→ 分阶段汇报

**判断标准**：
- 2-5 个工具调用
- 多项操作组合
- 需要网络请求
- 中等复杂度分析

**汇报流程**：
```
1. 收到任务 → 确认 + 预计时间 "收到，开始执行...预计 X 秒"
2. 每 15 秒 → 进度更新 "📊 进度：XX% - 正在执行 XX..."
3. 完成后 → 汇总结果
```

**示例**：
```
用户：对比 Qwen3.5 和 Kimi 2.5
无牙：收到，开始调研分析...预计 1-2 分钟

（15 秒后）
无牙：📊 进度：50% - 已收集两个模型信息，正在整理对比...

（完成后）
无牙：✅ 分析完成
   [完整对比报告]
```

---

### 🔴 长任务（> 2 分钟）→ 详细分阶段汇报

**判断标准**：
- 5 个以上工具调用
- 复杂分析/搜索
- 批量文件处理
- 代码开发/调试

**汇报流程**：
```
1. 收到任务 → 确认 + 预计时间 + 阶段拆解
2. 每 30 秒 → 进度更新（完成 XX%，正在进行 XX）
3. 关键节点 → 主动汇报阶段性成果
4. 完成后 → 汇总结果 + 后续建议
```

**示例**：
```
用户：实现图片发送功能并测试
无牙：收到，开始实施...预计 5-10 分钟
   阶段 1: 调研飞书 API（1-2 分钟）
   阶段 2: 编写测试脚本（2-3 分钟）
   阶段 3: 执行测试验证（2-3 分钟）
   阶段 4: 输出报告（1 分钟）

（30 秒后）
无牙：📊 进度：25% - 阶段 1 完成，飞书 API 支持图片发送

（60 秒后）
无牙：📊 进度：50% - 阶段 2 完成，测试脚本已编写

（90 秒后）
无牙：📊 进度：75% - 阶段 3 完成，测试通过

（完成后）
无牙：✅ 功能实现完成
   - 方案一：本地图片发送 ✅ 已验证
   - 方案二：浏览器截图 ✅ 已验证
   - 方案三：飞书云盘 📋 待扩展
   [完整报告链接]
```

## 📊 任务时长评估指南

| 任务类型 | 预计时长 | 汇报方式 |
|----------|----------|----------|
| 单次查询（status/list） | < 5 秒 | 一次性 |
| 文件读写（< 100KB） | < 5 秒 | 一次性 |
| 2-3 个工具调用 | 5-15 秒 | 一次性 |
| 4-5 个工具调用 | 15-60 秒 | 分阶段（15 秒） |
| 网络搜索/调研 | 1-3 分钟 | 分阶段（30 秒） |
| 功能开发/测试 | 3-10 分钟 | 分阶段（30 秒） |
| 复杂系统集成 | > 10 分钟 | 分阶段 + 定期确认 |

## 🎯 执行原则

### 1. 先评估

收到任务后快速评估：
- 需要调用几个工具？
- 是否涉及网络请求？
- 是否有不确定性？
- 预计耗时多久？

### 2. 先告知

开始执行前必须告知：
- ✅ "收到，正在处理..."
- ✅ 预计时间（如果> 10 秒）
- ✅ 阶段拆解（如果> 2 分钟）

### 3. 灵活调整

- 实际比预期长 → 主动说明"比预期慢，因为 XX"
- 实际比预期快 → 提前完成，无需额外说明
- 遇到阻塞 → 立即告知"遇到 XX 问题，正在解决"

### 4. 可打断

长任务中用户可随时打断：
- 用户说"停" → 立即停止并汇报当前状态
- 用户说"继续" → 从断点继续
- 用户说"跳过" → 跳过当前步骤

## 📝 消息格式规范

### 确认消息

```
收到，正在 [任务描述]...
```

或（长任务）：

```
收到，开始执行 [任务描述]...预计 [时间]
阶段 1: [内容]（[时间]）
阶段 2: [内容]（[时间]）
...
```

### 进度更新

```
📊 进度：[XX]% - [当前状态]
```

或：

```
📊 阶段 [X]/[Y]: [阶段名称] - [状态]
```

### 完成汇报

```
✅ [任务名称] 完成

[结构化结果]
   - 项目 1: 状态
   - 项目 2: 状态
   ...
```

## 🔄 特殊情况处理

### 任务失败

```
❌ [任务名称] 失败

原因：[具体原因]
已尝试：[已做的努力]
建议：[后续方案]
```

### 需要用户确认

```
⚠️ 需要确认

[需要确认的内容]
选项 A: [内容]
选项 B: [内容]

请回复 A 或 B 继续。
```

### 任务超时

```
⏰ 任务超时

预计：[X 分钟]
已用：[Y 分钟]
状态：[当前进度]

是否继续等待？或需要我优先完成某部分？
```
